## 本次更新概述

今天的更新主要集中在 Android 端，将之前的网页原型进一步整理成可在手机上运行的移动应用版本。核心工作包括：完善原生 Android 界面、接入语音识别流程、增加提词控制能力、补充设置项，并完成 debug APK 构建。

## 主要更新内容

1. 完善安卓端主流程

- 新增首页步骤式操作流程：设置识别密钥、测试麦克风、导入与设置、进入提词。
- 将提词器从简单页面推进为完整移动端使用流程。
- 增加进入提词模式和返回首页的状态切换。

2. 增强语音识别能力

- 接入 Android 原生 `SpeechRecognizer`。
- 增加麦克风权限申请与权限结果处理。
- 增加麦克风测试状态提示。
- 增加云端语音识别配置项：API Key、AppID、Secret Key。
- 增加基于 WebSocket 的实时语音识别流程。
- 保留普通云端识别兜底逻辑。

3. 完善提词跟读体验

- 支持开始、暂停、重置。
- 支持朗读进度匹配，并对已读文字变色。
- 支持当前朗读位置高亮。
- 支持前进、后退微调朗读位置。
- 支持自动滚动，卡住时可继续推进。
- 支持点击提词区开始或暂停。

4. 增加显示与跟读设置

- 增加跟读设置：变色速度、自动推进速度、滑动速度。
- 增加显示设置：字体大小、已读颜色、当前字颜色、背景颜色。
- 增加快捷设置入口，方便在提词模式中调整稿件和显示效果。
- 增加历史稿件保存和加载能力。

5. 优化横屏和全屏提词

- 增加横屏/竖屏切换。
- 增加全屏横屏提词模式。
- 横屏提词时隐藏不必要的首页面板，突出提词内容。

## 补充：百度语音技术实时语音识别开通步骤

这部分建议放在开发说明或部署说明中。对外 Release 文案仍建议使用“云端语音识别”“实时语音识别服务”等通用名称。

1. 登录百度智能云或百度 AI 开放平台。

- 进入百度智能云控制台或 AI 开放平台控制台。
- 使用已完成实名认证的账号登录。
- 如果账号未完成实名认证，需先按平台提示完成认证，否则可能无法创建应用或开通付费能力。

2. 进入语音技术服务。

- 在控制台中找到“人工智能”或“AI 服务”分类。
- 进入“语音技术”相关产品页面。
- 找到“语音识别”能力，并选择“实时语音识别”服务。

3. 创建应用。

- 在语音技术控制台中新建应用。
- 应用类型可按实际项目选择移动应用、Android 应用或通用应用。
- 应用名称可填写“语音跟读提词器”或项目内部名称。
- 创建完成后，进入应用详情页。

4. 获取识别密钥。

在应用详情中记录以下信息：

```text
AppID
API Key
Secret Key
```

这些信息用于应用内“设置识别密钥”页面。不要把真实密钥提交到 GitHub，也不要写入公开 README。

5. 开通实时语音识别能力。

- 在应用能力或产品服务配置中，确认已启用实时语音识别相关能力。
- 如果控制台要求单独领取免费额度、开通计费或同意服务协议，需要按页面提示完成。
- 如果项目同时使用普通语音识别兜底，也需要确认普通语音识别能力可用。
- 实时语音识别通常提供每月 10 小时免费额度，而不是仅首月免费。
- 按量付费价格为 3 元/小时。
- 以上额度与价格为截至 2026 年 6 月 23 日的信息，最终价格和计费规则以官方控制台与官方价格页为准。

6. 在手机应用中填写密钥并测试。

- 安装最新 debug APK。
- 打开应用，进入“设置识别密钥”。
- 填写 AppID、API Key、Secret Key。
- 返回首页，先执行“麦克风测试”。
- 进入提词模式，点击“开始”，朗读稿件内容。
- 如果识别正常，提词区会逐步高亮已读文字。

7. 常见检查项。

- 手机已允许麦克风权限。
- 手机网络正常。
- AppID 必须填写数字形式。
- API Key 和 Secret Key 不要填反。
- 控制台中应用状态正常，实时语音识别能力已开通。
- 如果提示识别失败，优先检查服务是否开通、密钥是否正确、账号是否欠费或额度是否用完。
